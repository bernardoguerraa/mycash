# How-tos

> Receitas práticas para resolver tarefas específicas. Para quem já conhece o básico.

How-tos assumem que o leitor sabe o que precisa fazer; só precisa do **caminho mais curto**.

## Conteúdo planejado

- `como-adicionar-rota-protegida.md` — criar uma rota dentro de `src/app/(dashboard)/`
- `como-criar-migration-supabase.md` — adicionar nova tabela ou coluna usando o CLI do Supabase
- `como-conectar-conta-pluggy.md` — fluxo de Open Finance via `PluggyConnectButton`
- `como-rodar-typecheck-localmente.md` — `npx tsc --noEmit`
- `como-fazer-deploy-vercel.md` — variáveis de ambiente e gatilhos de deploy

## Característica de um bom how-to

- Começa com a tarefa, não com conceitos
- Lista pré-requisitos no topo
- Termina quando a tarefa está concluída — sem teoria extra
