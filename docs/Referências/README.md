# Referências

> Especificações técnicas para consulta rápida. Informação descritiva e completa.

Referência é o que o leitor consulta quando já está trabalhando e precisa lembrar de um
detalhe — não é material de aprendizado.

## Conteúdo planejado

- `schema-banco.md` — tabelas Supabase (`profiles`, `accounts`, `transactions`, `goals`, `reminders`, `notifications`, integrações Pluggy)
- `rotas-api.md` — endpoints em `src/app/api/` (Pluggy connect-token, sync, webhook)
- `variaveis-ambiente.md` — `.env.local` com todas as variáveis e seus formatos
- `componentes.md` — catálogo de componentes em `src/components/`
- `tipos.md` — interfaces TypeScript em `src/types/database.ts`

## Característica de uma boa referência

- Estrutura previsível (sempre o mesmo formato por entrada)
- Neutra: descreve, não recomenda
- Completa: lista todas as opções, não só as comuns
