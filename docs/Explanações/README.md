# Explanações

> Discussão e contexto. Material para **entender** o projeto, não para executá-lo.

Explanações respondem perguntas como "por que essa stack?", "que alternativas
foram consideradas?", "qual é o trade-off dessa decisão?".

## Conteúdo

- [`00-sessao-desenvolvimento.md`](./00-sessao-desenvolvimento.md) — relatório da sessão de desenvolvimento inicial
- [`01-stack-frontend.md`](./01-stack-frontend.md) — escolha do framework de frontend
- [`02-implementacao-figma.md`](./02-implementacao-figma.md) — status da implementação das telas do Figma

## Característica de uma boa explanação

- Discute alternativas, não só a decisão final
- Exibe o contexto histórico (por que decidimos X **naquele momento**)
- Pode ser opinativa, ao contrário de Referência
