# Documentação — MyCash

Esta documentação segue o framework [**Diátaxis**](https://diataxis.fr), que organiza
conteúdo técnico em quatro categorias com objetivos distintos. Cada categoria responde
a uma pergunta diferente do leitor.

## Estrutura

```
docs/
├── Tutoriais/      → "Como começo?"           (aprender)
├── How-tos/        → "Como faço X?"           (executar tarefa)
├── Referências/    → "O que é X?"             (consultar)
└── Explanações/    → "Por que decidimos X?"   (entender)
```

## Quando usar cada categoria

| Categoria      | Foco             | Forma             | Analogia            |
|----------------|------------------|-------------------|---------------------|
| Tutoriais      | Aprendizado      | Passo a passo     | Aula prática        |
| How-tos        | Tarefa concreta  | Receita           | Receita de cozinha  |
| Referências    | Informação       | Especificação     | Dicionário          |
| Explanações    | Compreensão      | Discussão         | Artigo de revista   |

## Índice

- [Tutoriais](./Tutoriais/) — onboarding, primeiros passos
- [How-tos](./How-tos/) — receitas para tarefas comuns
- [Referências](./Referências/) — schema, rotas, variáveis de ambiente
- [Explanações](./Explanações/) — decisões de stack e arquitetura
